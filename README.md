# 🏥 Healthcare Management System

A robust solution designed to streamline the lifecycle of patient appointments through **role-based access control**, **automated QR-based check-ins**, and **centralized patient management**.

---

### 🚀 Project Goal
To digitize the clinic experience by:
*   **Efficiency**: Allowing staff to efficiently register patients and book slots.
*   **Real-time**: Enabling doctors to manage their consultation queues in real-time.
*   **Oversight**: Providing admins with total oversight of the system's data and users.

---

### 🛠 Core Modules

| Module | Description |
| :--- | :--- |
| **Users & Authentication** | Secure management of Staff, Doctor, and Admin accounts. |
| **Patient Management** | Centralized database for patient registration and medical history. |
| **Appointment Engine** | Advanced scheduling with real-time status tracking. |
| **QR Check-in System** | Contactless patient arrival verification at the counter. |
| **Admin Console** | Auditing, reporting, and high-level system configurations. |

---

### 🔄 The Lifecycle
The system follows a linear progression for every appointment:

1.  **Registration**: Staff onboard a new patient.
2.  **Booking**: Appointment is scheduled; a unique QR code is generated.
3.  **Arrival**: Patient scans the QR code at the counter to "Check-In."
4.  **Consultation**: Doctor sees the patient in the queue and begins the session.
5.  **Resolution**: Session is marked as complete with clinical notes recorded.

---

### 👥 Stakeholders

*   **👑 Admins**: System owners with full CRUD permissions across the entire platform.
*   **📋 Staff**: Operational users focused on registration, booking, and front-desk logistics.
*   **🩺 Doctors**: Clinical users focused on patient care, consultation updates, and medical history.
